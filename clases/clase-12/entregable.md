# Plan de Implementación de 7 Días
## Tu sistema de Growth con Claude operando completamente esta semana

**Módulo 04 · Clase 12 · Claude para Growth**

---

Este es tu roadmap de implementación. No una lista de pendientes — un plan con días específicos, acciones concretas y criterios de éxito para saber si cada paso funcionó antes de pasar al siguiente.

Tiempo diario estimado: **30–60 minutos por día**.
Resultado al final del Día 7: sistema de Growth con Claude completamente operativo.

## DÍA 1 — Fundamentos: tu base de operaciones activa

**Objetivo del día:** Claude trabaja con el contexto real de tu negocio desde hoy.

**Tiempo estimado: 45 minutos**

---

### Acciones del Día 1

**Acción 1.1 — Cuenta y acceso (10 min)**
- [ ] Confirmar que tienes cuenta Pro activa en Claude.ai
- [ ] Descargar la app de escritorio si no la tienes (claude.ai/download)
- [ ] Añadir Claude al dock (Mac) o a la barra de tareas (Windows)
- [ ] Crear acceso directo en el navegador a tu Project principal

> Si ya tienes todo esto: pasa directo a la Acción 1.2.

---

**Acción 1.2 — Revisar o crear tu Project principal (15 min)**

Si ya tienes un Project creado, ábrelo y revisa las instrucciones de sistema.
Si no tienes Project, crea uno ahora. Nombre: `[Tu empresa] — Growth OS`.

Usa esta plantilla de instrucciones de sistema como base (o para actualizar las que tienes):

```
Eres el co-piloto de Growth de [NOMBRE DE TU EMPRESA].

ROL Y CONTEXTO:
Mi rol: [Chief of Growth / Founder / Head of Growth / otro]
Etapa del negocio: [early stage / PMF / growth / scale]
Equipo de Growth: [solo yo / equipo de X personas]

EL NEGOCIO:
Producto: [qué hace, en 2 líneas]
Modelo de negocio: [SaaS / eCommerce / marketplace / servicios / otro]
Precio/Ticket promedio: [monto]
MRR o facturación mensual actual: [cifra o rango]

CLIENTE IDEAL (ICP):
Perfil: [rol, tipo de empresa, tamaño]
Dolor principal que resuelve el producto: [descripción concreta]
Cómo describe ese dolor con sus propias palabras: [cita o paráfrasis]

MÉTRICAS QUE IMPORTAN:
Las métricas que monitoreo semanalmente: [lista — ej: MRR, CAC, churn, tasa de activación, NPS]
Benchmark de mi industria (si lo sé): [ej: churn SaaS B2B < 2% mensual]

FOCO ACTUAL:
El área de mayor prioridad este trimestre: [adquisición / activación / retención / revenue]
El principal cuello de botella en el funnel hoy: [descripción]

CÓMO QUIERO QUE RESPONDAS:
- Directo y concreto. Sin relleno. Sin adjetivos innecesarios.
- Prioriza acciones que se pueden ejecutar esta semana.
- Si necesitas más contexto para dar una respuesta útil, pídelo antes de responder.
- Usa bullets cuando listes cosas. Usa tablas cuando compares opciones.
- Si detectas una contradicción en lo que te pido, señálala antes de responder.
```

---

**Acción 1.3 — Prueba de alineación (10 min)**

Con las instrucciones de sistema cargadas, escribe este prompt en el chat del Project:

```
¿Qué entiendes sobre mi negocio, mi rol y el foco actual basándote en el contexto que tienes?
Dime también qué información adicional sería más útil que cargues para darte respuestas más precisas.
```

Verifica:
- [ ] Menciona tu negocio correctamente
- [ ] Entiende tu rol y etapa
- [ ] Identifica el foco actual (adquisición / activación / etc.)
- [ ] Habla en el tono correcto: directo, concreto, sin relleno
- [ ] Te pide información adicional relevante (ese es el contexto que cargas en el Día 2)

Si algún punto está mal: ajusta las instrucciones de sistema y repite.

---

**Acción 1.4 — Primer output real del día (10 min)**

Haz este prompt adaptado a tu situación actual:

```
Dame los 3 experimentos que priorizarías esta semana basándote en lo que sabes de mi negocio.
Para cada uno: hipótesis, variable a testear, criterio de éxito y esfuerzo estimado.
```

Evalúa: ¿el output refleja tu contexto real o es genérico? Si es genérico, las instrucciones de sistema necesitan más detalle.

---

**Criterio de éxito del Día 1:**
Claude responde la prueba de alineación con tu negocio específico (no genérico) y los 3 experimentos propuestos tienen sentido para tu situación actual. Si lo pasan: el Día 1 está completo.

## Señales de que el sistema está funcionando

Al final de la primera semana completa de operación, deberías ver:

**Señales positivas:**
- Empiezas el día con más contexto sobre tu negocio del que tenías antes
- Tomas decisiones con datos en menos tiempo
- Produces copies o análisis en minutos, no en horas
- El equipo (si lo tienes) usa los skills sin preguntarte cómo

**Señales de que algo necesita ajuste:**
- Los outputs de Claude son genéricos o podrían ser de cualquier empresa
  → Ajusta las instrucciones de sistema: necesitan más contexto específico
- Los agentes salen de su área y hablan de todo
  → Refuerza las restricciones en el system prompt
- Los skills producen outputs que no usas
  → El skill no está bien definido para tu caso — ajusta el "Cuándo usar" y el prompt
- No estás abriendo Claude todos los días
  → El problema no es el sistema — es la rutina. Define un momento fijo diario (ver Clase 01, Paso 10)
